# jj perf

## why is it so slow?

### _jj_ performance

I've been using [jj](https://jj-vcs.dev)
for a while as my preferred git frontend.

But using it on the Go repo was lagging...
0.5s to run `jj status`.

After being torubled by it for a bit too long,
I finally figured I should look into it.
[debug logs with `JJ_LOG=debug`](https://docs.jj-vcs.dev/latest/contributing/#logging)
weren't all that helpful.

[tracing with `JJ_TRACE_INCLUDE_ARGS=1 JJ_TRACE=/tmp/trace.json`](https://docs.jj-vcs.dev/latest/contributing/#profiling)
and viewing the results on [ui.perfetto.dev](https://ui.perfetto.dev/)
showed me that it was making an absurd number of `read_commit` calls.
And then it hit me,
I should bisect by config file...

Turns out the culprit was this line,
specifically the `latest(mine(), 3)` part.
Removing it made it much faster.

```toml
[revsets]
log = "ancestors(trunk() | @ | working_copies() | latest(mine(), 3), 5)"
```

`revsets.short-prefixes` which is used to calculate set of changes that
can have shorter IDs (single letter, etc) defaults to the `log` revset
if unset.
And `jj` seems to use it a lot, so it's slow.
